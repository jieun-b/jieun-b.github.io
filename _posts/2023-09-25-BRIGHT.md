# BRIGHT: Bi-level Feature Representation of Image Collections using Groups of Hash Tables

> arXiv 2023. [[Paper](https://arxiv.org/abs/2305.18601)] [[Github](https://bright-project01.github.io/)]
> 

![img1.png](/assets/img/2023-09-25/img1.png)

# Introduction

본 논문은 Instant NGP의 모든 이점은 유지하면서, 단일 이미지가 아닌 이미지 컬렉션에 대한 feature representation을 수행하는 것을 목적으로 한다. 동시에, SOTA generator를 사용하여 이미지 코드에 대한 생성 모델링을 가능하게 한다.

BRIGHT는 이미지 컬렉션을 위한 bi-level feature representation으로, multi-scale feature grid 공간 위에 이미지 별 latent space를 가지는 구조로 구성되어있다. 이는 오토인코더가 이미지 컬렉션을 표현하도록 학습되면서 이루어진다. 먼저 이미지 레벨에서 이미지 컬렉션을 연속적인 키 코드로 인코딩한다. 이 때 각 이미지 하나 당 하나의 키 블록을 가진다. Feature 레벨에서는 키 코드를 사용하여 multi-resolution hash table 집합에서 feature를 검색한다. 검색된 feature는 작은 MLP를 통과하여 StyleGAN 유형의 디코더로 들어가 입력 이미지를 복원하게 된다. 

Discrete Vector Quantization(VQ)에 기반한 생성 모델은 종종 코드북의 낮은 사용률이 문제가 된다. 이와 비교하여 BRIGHT의 키 코드와 해시 테이블은 gradient flow와 연속적으로 함께 훈련되어, 해시 테이블 항목들의 높은 사용률과 개선된 생성 모델링을 가능하게 한다. 키 코드는 기존의 continuous representation과 달리 엄격하게 크기와 분산에 제한이 있다. 그러나 BRIGHT에서 학습한 이미지 코드는 정밀도가 높고 Latent Diffusion Models (LDMs)의 denoising process에 적합하지 않기 때문에 학습 중에 노이즈를 주입하는 과정을 거치게 된다.

# Method

데이터셋은 오토인코더 프레임워크에서 압축된 키 코드와 해시 테이블 집합을 이용하여 표현될 수 있다. 이미지 특징 공간은 해시 테이블에서 간결하고 구조적으로 구성된다. 어떤 이미지든지 키 코드를 참조하고 해시 테이블에서 관련된 특징 벡터를 보간하여 인코딩 할 수 있다.

전체 구조는 다음과 같이 세 부분으로 구성되어 있다: Encoding, Hash Retrieval, Decoding. 

![img2.png](/assets/img/2023-09-25/img2.png)

## Encoding

차원이 $H×W$이고 channel width가 $C$인 이미지 $I∈\mathbb{R}^{H×W×C}$가 주어질 때, 인코더 네트워크는 이를 연속적인 공간에서 키 코드 $z∈[0,1]^{H_{z}×W_{z}×C_{z}}$로 변환한다. 여기서 $C_{z}=N_{h}×C_{key}$이며 $N_{h}$는 해시 테이블 그룹의 수, $C_{key}$는 각 해시 테이블에 해당하는 키 유닛의 길이이다. 인코더는 시그모이드 함수를 추가하여 $z$를 $[0,1]$로 정규화하는 것을 제외하고는 VQGAN의 디자인을 따른다. 이러한 연속적인 표현 $z$는 키로 사용되어 multi-resolution hash table에 접근할 수 있도록 한다. 다중 해상도 해시 테이블은 전체 이미지 데이터셋에 대한 이미지 feature의 그리드 기반 표현을 제공한다.

## Hash Retrieval

Multi-resolution hash table은 해상도에 따라 여러 레벨을 가진 테이블로 구성된다. 이러한 테이블은 spatial hash function을 통해 인덱싱 되는데, 이는 인코딩된 키와 공간적으로 이웃한 항목을 검색하기 위함이다. 검색된 항목은 선형 보간으로 주어진 키에서 이미지 특징을 얻는다. 

먼저 연속 코드 $z$를 교차로 타일링하여 사이즈가 ${H_{d}×W_{d}×C_{z}}$인 텐서 $\hat{z}$를 형성한다. 타일링은 디코더로 보내질 특징의 공간적 크기를 확장하기 위해 사용된다. $\hat{z}$의 각 그리드 $$\hat{z}_{i,j}$$에 대해 채널 $C_{z}$를 $N_{h}$개의 조각으로 나눈다. 각각의 코드 조각들은 $1×1×C_{key}$ 크기를 가지게 된다. 계산 복잡도로 인해 $C_{z}$차원의 키를 쿼리로 사용하는 해시 테이블 구성이 불가능하기 때문에 하나의 해시 테이블을 $C_{key}$차원의 각각의 조각에 별도로 할당한다. 각각의 코드 조각 $$\hat{z}_{i,j}^{s}(1\le s\le N_{h})$$에 작은 노이즈 $\epsilon_{o}∼\mathcal{N}(0,\frac{1}{(4r_{max})^{2}})$를 주입한다. $r_{max}$는 다중 해상도 해시 테이블 그룹의 최대 해상도를 의미한다. 왜곡된 $$\hat{z}_{i,j}^{s}$$는 결합된 키 코드를 형성하기 위해 정규화된 공간 좌표$(x=\frac{i-0.5}{H_{d}},y=\frac{i-0.5}{W_{d}})$와 연결된다. 

![img3.png](/assets/img/2023-09-25/img3.png){: width="60%" height="60%"}

여기서 $s$는 조각의 인덱스이며 $q_{i,j}^{s}∈[0,1]^{C_{key}+2}$이다. 작은 왜곡은 $$\hat{z}_{i,j}^{s}$$의 안정된 데이터 정밀도를 위한 것이며 다음 단계에서 diffusion 학습을 완화시킬 것이다. 키 코드에 $(x,y)$를 통합하는 것은 다중 해상도 해시 테이블에서 spatial-aware feature를 검색하도록 한다. 검색된 특징 $f_{i,j}^{s}$는 특징 블록 $f∈\mathbb{R}^{H_{d}×W_{d}×C_{d}}$의 대응하는 위치에 배치되며 디코더의 입력 특징으로 사용된다. 수학적으로 $f_{i,j}^{s}$는 다음과 같이 얻어진다.

![img4.png](/assets/img/2023-09-25/img4.png){: width="60%" height="60%"}

![img5.png](/assets/img/2023-09-25/img5.png)

$F_{r}$은 해시 검색 함수를 나타내며 해시 테이블에서 키 코드 $q_{i,j}^{s}$를 포함하는 그리드 노드를 반환한다. 결과는 $(f_{k}^{v},c_{k}^{v})$의 집합으로, $f^{v}∈\mathbb{R}^{N_{r}×C_{v}}$는 특징 벡터, $c^{v}∈[0,1]^{C_{key}+2}$는 좌표이다. $N_r$은 해시 테이블 그룹의 해상도 개수이며 $C_{v}$는 해시 테이블에서의 특징 벡터 차원이다. $F_r$은 다음과 같이 정의할 수 있다.

![img6.png](/assets/img/2023-09-25/img6.png){: width="60%" height="60%"}

$M=2^{C_{key}+2}$이다. 만약 $C_{key}=1$이라면 $c_{k}^{v}$는 큐브의 좌표를, $f_{k}^{v}$는 각 큐브의 노드와 관련있는 8개의 특징 벡터를 나타낼 것이다. $F_{inp}$는 $N_{r}×C_{v}$ 크기의 특징 벡터를 반환하는 $f_{k}^{v}$들 사이의 선형 보간을 나타낸다. 

![img7.png](/assets/img/2023-09-25/img7.png){: width="60%" height="60%"}

$F_d$는 $c_{k}^{v}$에서 $q_{i,j}^{s}$까지 거리 함수로 특징 벡터 $f_{k}^{v}$의 가중치를 결정한다. $C_{key}=1$일 때, $F_{inp}$는 큐브의 노드에 대한 삼선 보간이다. 마지막으로 $F_{mlp}$는 MLP를 사용하여 보간된 특징 벡터의 차원을 $N_{r}×C_{v}$에서 $C_{d}/N_{h}$로 매핑한다.

## Decoding

디코더의 구조는 StyleGAN의 설계를 따른다. 디코더는 $f$에서 average pooling과 MLP를 통해 계산된 style code $$∈\mathbb{R}^{1×1×C_{d}}$$에 의해 모듈레이션 된다. 전체 오토인코딩 과정은 $$\mathcal{L}_{2}$$ loss와 perceptual loss $$\mathcal{L}_{percep}$$, Wasserstein GAN loss $$\mathcal{L}_{WGAN}$$, 판별자의 gradient penalty $$\mathcal{L}_{gp}$$를 사용해 학습된다. 전체 loss는 다음과 같고 인코더와 디코더, 해시 테이블을 공동으로 최적화된다.

## Properties

다음으로 모델의 성능 향상에 도움이 되는 네 가지 속성에 대해 논의한다.

![img9.png](/assets/img/2023-09-25/img9.png)

### Continuity

VQ와 달리 논문에서는 키 코드와 해시 테이블을 연속적으로 함께 훈련하며 명확한 그래디언트 플로우와 함께 학습된다. 그림 5(a)의 2D 그리드와 같이 입력 키 코드 $(x,y)$에서 검색된 값은 이를 둘러싸고 있는 네 개의 그리드 노드로 선형 보간된다. 방향은 $∆x=[(y_{2}-y)(Q_{4}-Q_{3})+(y-y_{1})(Q_{2}-Q_{1})]\cdot grad$ 로 이동된다. ($Q_{i}$는 노드의 값이다.) 그림 5(b)는 해시 테이블의 모든 항목들이 검색되며 대부분의 해시 테이블 그룹에서 분산이 작다는 것을 보여준다. 이 결과는 다중 해상도 해시 테이블의 항목 사이에 적중하는 정도가 균일하게 분포한다는 것을 시사한다.

### Relaxed Data Precision

이미지 생성을 위해서는 키 코드에 확산을 적용해야 한다. Diffusion process는 RGB 값에 대해 잘 작동하기 때문에 LDM에서는 이를 latent code에 적용하기 위해 regularization을 사용했다. 본 논문에서는 연속적인 키 코드를 사용하기 때문에 키 코드의 정밀도를 완화함으로써 더 날카롭고 정확한 데이터를 생성할 수 있는 것을 관찰했다. 이는 그림 5(c)의 단순화된 1D 그리드와 같이, 노이즈 $\epsilon_{o}∼\mathcal{N}(0,\frac{1}{(4r_{max})^{2}})$를 $$\hat{z}_{i,j}^{s}$$에 주입함으로써 수행된다. 학습 후에 오토인코더는 왜곡된 키 코드가 $$\hat{z}_{i,j}^{s}\pm\frac{1}{2r_{max}}$$의 범위 내에 있는 한 복원 결과를 잘 생성하도록 한다. 표준 편차를 $\epsilon_{o}$에서 $\frac{1}{4r_{max}}$로 세팅함으로써 왜곡된 $$\hat{z}_{i,j}^{s}$$에 대해 검색된 특징 벡터는 해시 테이블의 세 개의 인접한 노드 내에서 다양하게 변화한다. 편차 값은 diffusion process의 유연성과 오토인코더에서 과도한 불확실성을 피하는 것 사이에서 균형을 맞춘다.

### Strictly Bounded Variance

Diffusion 모델 학습 시에, element-wise rescaling 방식이 KL-regularized latent code에 사용된다. 여기서 rescaling factor는 훈련 데이터의 첫 번째 미니 배치에서 계산된다. 이와 비교하여 논문에서의 연속적인 latent code는 $[0,1]$의 범위에 있는 해시 테이블의 키다. 따라서 엄격하게 분산에 제한을 가진다.

### Auxiliary Data Structure

인코딩된 특징을 디코더에 직접 통과시키는 대신, 다중 해상도 해시 테이블과 같은 보조 데이터 구조로부터 특징 검색을 위한 키를 인코딩한다. 다중 해상도 해시 테이블은 다중 해상도 해시 그리드와 보편적인 해싱을 통해 강력한 feature composition 능력을 제공한다.
